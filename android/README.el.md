# AccessEye για Android

[Read in English](README.md)

Η έκδοση του AccessEye για Android: το ίδιο προϊόν με την εφαρμογή iOS στο
`../ios/`, ένας offline αφηγητής σκηνών με τεχνητή νοημοσύνη που τρέχει στη
συσκευή, για τυφλούς χρήστες και χρήστες με μειωμένη όραση. Στρέφετε το
κινητό, αγγίζετε οπουδήποτε, και σας περιγράφει φωναχτά τι υπάρχει μπροστά
σας, στη γλώσσα σας. Χωρίς internet, χωρίς λογαριασμό, χωρίς cloud.

## Stack

| Τομέας    | Τι χρησιμοποιεί                                          |
|-----------|----------------------------------------------------------|
| UI        | Kotlin + Jetpack Compose (Material 3), edge to edge      |
| Μοντέλο   | Gemma 3n E2B (int4), ακριβώς το ίδιο αρχείο `.litertlm` που κατεβάζει η εφαρμογή iOS |
| Inference | LiteRT-LM Android API (Google AI Edge), αποκωδικοποίηση στη CPU + vision encoder στη GPU |
| Κάμερα    | CameraX                                                   |
| Ομιλία    | Android `TextToSpeech`, η καλύτερη εγκατεστημένη offline φωνή ανά γλώσσα |
| A11y      | TalkBack live regions, δονήσεις, UI που εκφωνεί μόνο του, RTL για τα αραβικά |

Η εφαρμογή εκφωνεί μόνη της κάθε αλλαγή κατάστασης στη γλώσσα που έχει
επιλέξει ο χρήστης, οπότε είναι πλήρως χρηστική και με το TalkBack κλειστό.
Όταν το TalkBack τρέχει, τα σύντομα μηνύματα κατάστασης περνούν μέσα από
αυτό, ώστε να μην μιλούν ποτέ δύο φωνές ταυτόχρονα. Η ίδια η περιγραφή της
σκηνής χρησιμοποιεί πάντα τη δική της φωνή της εφαρμογής, στην επιλεγμένη
γλώσσα.

## Η λήψη του μοντέλου

Η εφαρμογή είναι μικρή (περίπου 48 MB). Στην πρώτη εκκίνηση κατεβάζει το
μοντέλο Gemma 3n μία φορά (περίπου 3,4 GB) και μετά δουλεύει πλήρως offline.
Η λήψη τρέχει σε foreground service με ειδοποίηση προόδου, οπότε συνεχίζει
ακόμα κι αν βγείτε από την εφαρμογή ή κλειδώσετε την οθόνη, και συνεχίζει από
εκεί που σταμάτησε αν πέσει η σύνδεση (HTTP range requests πάνω σε αρχείο
`.part`). Μπορείτε να διαγράψετε και να ξανακατεβάσετε το μοντέλο όποτε
θέλετε από τις Ρυθμίσεις.

Το URL λήψης βρίσκεται στο
`app/src/main/java/gr/orestislef/accesseye/ai/AppConfig.kt` και δείχνει στο
ίδιο αρχείο που χρησιμοποιεί η εφαρμογή iOS, ώστε οι δύο πλατφόρμες να
μένουν συγχρονισμένες.

## Πώς να το χτίσετε

Απαιτήσεις: JDK 17+, Android SDK 37, και μια συσκευή με 8 GB RAM για το
πραγματικό μοντέλο (δοκιμασμένο σε Moto G15, Android 15).

```
cd android
./gradlew :app:assembleDebug
./gradlew :app:installDebug
```

Τα release builds υπογράφονται αυτόματα όταν υπάρχει το
`signing/keystore.properties` (βλ. παρακάτω). Διαφορετικά, το
`assembleRelease` παράγει ανυπόγραφο build.

```
./gradlew :app:assembleRelease   # APK
./gradlew :app:bundleRelease     # AAB for Play
```

### Υπογραφή

Δημιουργήστε ένα keystore μία φορά και περιγράψτε το στο
`signing/keystore.properties` (και τα δύο είναι στο gitignore, μην τα κάνετε
ποτέ commit):

```
storeFile=signing/accesseye-release.jks
storePassword=...
keyAlias=accesseye
keyPassword=...
```

### Δοκιμή χωρίς το μοντέλο

Ορίστε `useRealGemma = false` στο `AppConfig.kt` και η εφαρμογή τρέχει από
άκρη σε άκρη με έναν ενσωματωμένο demo περιγραφέα, χωρίς να χρειάζεται λήψη.
Βολικό για emulators.

## Γλώσσες

Αγγλικά, ελληνικά, ισπανικά, γαλλικά, γερμανικά, αραβικά, χίντι, ιταλικά και
ρωσικά. Ένα μοντέλο αναλαμβάνει και την περιγραφή και τη μετάφραση. Η αλλαγή
γλώσσας αλλάζει μόνο το prompt και τη φωνή εκφώνησης. Στην πρώτη εκκίνηση η
εφαρμογή ακολουθεί τη γλώσσα συστήματος του κινητού, εφόσον είναι μία από τις
εννέα. Αν η offline φωνή για κάποια γλώσσα δεν είναι εγκατεστημένη, οι
Ρυθμίσεις προσφέρουν συντόμευση για την εγκατάστασή της.

## Σημειώσεις επιδόσεων

Το Gemma 3n E2B χρειάζεται συσκευή της κατηγορίας των 8 GB RAM. Η παραγωγή
κειμένου τρέχει στη CPU (σε μεσαίας κατηγορίας Mali GPU αυτό είναι και
γρηγορότερο και πιο σταθερό από την αποκωδικοποίηση στη GPU) και ο vision
encoder τρέχει στη GPU, κάτι που το Gemma 3n απαιτεί. Η πρώτη περιγραφή μετά
από καθαρή εγκατάσταση είναι αργή όσο η μηχανή μεταγλωττίζει και αποθηκεύει
στην cache τα kernels της. Οι επόμενες ξαναχρησιμοποιούν την cache. Όσο το
μοντέλο σκέφτεται, η κάμερα απελευθερώνεται ώστε το UI να μένει αποκριτικό σε
αδύναμα chips.
